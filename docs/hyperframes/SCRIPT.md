# RELAY HyperFrames Script

Duration target: 60 seconds for a tight product tour. The Kaggle edit can expand this to 3 minutes with architecture and evaluation details.

## Narration

During a wildfire evacuation, care can break quietly.

Medication pickup, oxygen batteries, infant formula, transport questions, and unsafe health claims arrive as scattered texts and shelter notes.

RELAY groups those reports into a Care Continuity Ledger.

Medication continuity. Power-dependent care. Infant supplies. Mobility transport. Public updates.

Each ledger item keeps its source reports, missing fields, and unsafe claims visible.

This medication item looks urgent, but it is not ready.

Recipient identity, pickup contact, and pharmacy location are still missing.

One report also includes unsafe insulin dosing language, so RELAY suppresses it as medical advice.

Handoff stays unavailable.

RELAY can request missing information and flag review. It cannot mark the item ready while required fields or unsafe claims remain unresolved.

For the public preview, replay mode keeps the story reliable.

For technical verification, Ollama mode runs Gemma locally and validates the same triage schema.

RELAY keeps care continuity visible: what is linked, what is missing, what is unsafe, and why handoff is unavailable.
