# Design System - RELAY

## Direction
RELAY should read as enterprise incident operations software: calm, dense, reliable, and review-first. It is not a landing page, poster, or themed dashboard.

## Visual Rules
- Use flat neutral surfaces: off-white workspace, white panels, cool gray borders.
- Use one primary blue for action and focus. Use amber, red, and green only for status.
- Keep radius tight: 3px for panels, inputs, and buttons. Pills only for compact status tags.
- Avoid decorative shadows, gradients, glowing effects, and large rounded cards.
- Custom SVG assets should explain workflow, records, routing, or verification. They should not be mascot-style decoration.

## Layout Rules
- Prioritize workspace hierarchy: command bar, source reports, ledger, review dock.
- Prefer tables, rows, dividers, compact metadata, and real labels over card mosaics.
- Keep copy direct and operational. Say what is held, missing, verified, or ready.

## Current Custom SVG
- `apps/web/public/brand/relay-mark.svg` is a restrained operations mark: record sheet, shelter roof, check, and side routing rails.
